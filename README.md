## 파일 구조

### ✍️ Commit Convention
**형식**: `[타입]: 설명 #이슈번호`  
**예시**: `feat: 설명 #12`

| 타입     | 사용 시나리오 |
|----------|---------------|
| `feat`   | 신기능 추가 |
| `fix`    | 버그 수정 |
| `refactor`| 리팩토링 |
| `chore`  | 설정 변경 |
| `docs`	 | 문서 수정에 대한 커밋 |


### 🔄 Git Flow
| 브랜치 유형 | 목적 | 병합 대상 |
|-------------|------|------------|
| `main` | 프로덕션 버전 | 직접 커밋 금지 |
| `develop` | 개발 통합 브랜치 | feature/fix 브랜치 병합 |
| `feature/#/내용` | 기능 개발 | → develop |
| `hotfix/#/내용` | 긴급 수정 | → main & develop |
